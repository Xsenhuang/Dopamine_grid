## 格子布局设置功能

### 功能概述
允许用户自定义 Bingo 格子的布局大小，支持 3×3、4×4、5×5、6×6 四种布局。

### 使用场景
- 任务数量较少时选择小格子（如 3×3 = 9个任务）
- 任务数量较多时选择大格子（如 6×6 = 36个任务）
- 根据个人喜好调整界面密度

### 操作步骤
1. 点击首页底部的 **"任务"** 按钮进入任务管理页面
2. 在 **"格子布局设置"** 卡片中选择想要的布局
3. 点击 3×3、4×4、5×5 或 6×6 按钮
4. 如果当前任务数超过新布局容量，会提示是否裁剪

### 数据存储
- 布局设置保存在本地存储中，键名为 `grid_size`
- 默认布局为 5×5（25个格子）

### 注意事项
- 切换布局时，超出新布局容量的任务会被隐藏（不会删除）
- 切换回大布局时，之前隐藏的任务会重新显示
- 每个布局对应的最大任务数：
  - 3×3 = 9个任务
  - 4×4 = 16个任务
  - 5×5 = 25个任务
  - 6×6 = 36个任务

## 格子分数计算功能

### 功能概述
采用 Bingo 游戏规则计算完成进度，基于行、列、对角线的完成度来计算总体进度。

### 计算规则

#### 1. 线条定义
- **行（Row）**：横向一排格子
- **列（Column）**：纵向一排格子
- **对角线（Diagonal）**：斜向一排格子（左上到右下、右上到左下）

#### 2. 单线条完成度计算
```
单线条完成度 = (该线条已完成格子数 / 该线条总格子数) × 100%
```

**示例（4×4 布局）：**
- 一行有 4 个格子
- 完成 1 个 = 25%
- 完成 2 个 = 50%
- 完成 3 个 = 75%
- 完成 4 个 = 100%

#### 3. 总体进度计算
```
总体进度 = 所有线条完成度的平均值

线条总数 = 行数 + 列数 + 对角线数
        = n + n + 2
        = 2n + 2

其中 n 为布局大小（如 4×4 则 n=4）
```

**示例（4×4 布局）：**
- 行数：4 行
- 列数：4 列
- 对角线：2 条（主对角线 + 副对角线）
- 总线条数：4 + 4 + 2 = 10 条
- 总体进度 = 10 条线条完成度的平均值

### 进度显示
- **首页进度卡片**：显示总体完成百分比
- **进度条**：可视化展示完成进度
- **奖励弹窗**：显示完成度最高的线条信息

### 进度提示文案
根据总体进度显示不同的鼓励文案：
- 0-24%："完成一项继续加油 💪"
- 25-49%："不错的开始！继续 💪"
- 50-74%："已经完成一半了！✨"
- 75-99%："即将完成，加油！🔥"
- 100%："太棒了！今日任务全部完成 🎉"（显示庆祝动画）

### 特殊功能
- **完成庆祝**：当总体进度达到 100% 时，显示庆祝弹窗
- **线条详情**：在奖励弹窗中显示完成度最高的行/列/对角线

## 技术说明

### 本地存储键名
- `adhd_tasks` - 任务数据
- `grid_size` - 布局大小设置

### 数据结构
```javascript
// 任务对象
{
  id: 时间戳,
  name: "任务名称",
  color: "背景颜色",
  completed: false
}

// 线条进度对象
{
  index: 线条索引,
  completed: 已完成数,
  total: 总格子数,
  percent: 完成百分比
}
```
