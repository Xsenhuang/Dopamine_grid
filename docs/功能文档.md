## 格子布局设置功能

### 功能概述
允许用户自定义 Bingo 格子的布局大小，支持 3×3、4×4、5×5、6×6 四种布局。

### 使用场景
- 任务数量较少时选择小格子（如 3×3 = 9个任务）
- 任务数量较多时选择大格子（如 6×6 = 36个任务）
- 根据个人喜好调整界面密度

### 操作步骤
1. 点击首页底部的 **"任务"** 按钮进入任务管理页面
2. 在 **"格子布局设置"** 卡片中选择想要的布局
3. 点击 3×3、4×4、5×5 或 6×6 按钮
4. 如果当前任务数超过新布局容量，会提示是否裁剪

### 数据存储
- 布局设置保存在本地存储中，键名为 `grid_size`
- 默认布局为 5×5（25个格子）

### 注意事项
- 切换布局时，超出新布局容量的任务会被隐藏（不会删除）
- 切换回大布局时，之前隐藏的任务会重新显示
- 每个布局对应的最大任务数：
  - 3×3 = 9个任务
  - 4×4 = 16个任务
  - 5×5 = 25个任务
  - 6×6 = 36个任务

## 格子分数计算功能

### 功能概述
采用 Bingo 游戏规则计算完成进度，基于行、列、对角线的完成度来计算总体进度。

### 计算规则

#### 1. 线条定义
- **行（Row）**：横向一排格子
- **列（Column）**：纵向一排格子
- **对角线（Diagonal）**：斜向一排格子（左上到右下、右上到左下）

#### 2. 单线条完成度计算
```
单线条完成度 = (该线条已完成格子数 / 该线条总格子数) × 100%
```

**示例（4×4 布局）：**
- 一行有 4 个格子
- 完成 1 个 = 25%
- 完成 2 个 = 50%
- 完成 3 个 = 75%
- 完成 4 个 = 100%

#### 3. 总体进度计算
```
总体进度 = 所有线条完成度的平均值

线条总数 = 行数 + 列数 + 对角线数
        = n + n + 2
        = 2n + 2

其中 n 为布局大小（如 4×4 则 n=4）
```

**示例（4×4 布局）：**
- 行数：4 行
- 列数：4 列
- 对角线：2 条（主对角线 + 副对角线）
- 总线条数：4 + 4 + 2 = 10 条
- 总体进度 = 10 条线条完成度的平均值

### 进度显示
- **首页进度卡片**：显示总体完成百分比
- **进度条**：可视化展示完成进度
- **奖励弹窗**：显示完成度最高的线条信息

### 进度提示文案
根据总体进度显示不同的鼓励文案：
- 0-24%："完成一项继续加油 💪"
- 25-49%："不错的开始！继续 💪"
- 50-74%："已经完成一半了！✨"
- 75-99%："即将完成，加油！🔥"
- 100%："太棒了！今日任务全部完成 🎉"（显示庆祝动画）

### 特殊功能
- **完成庆祝**：当总体进度达到 100% 时，显示庆祝弹窗
- **线条详情**：在奖励弹窗中显示完成度最高的行/列/对角线

## 日历功能

### 功能概述
提供日历视图，用户可以查看任意日期的任务完成情况，包括完成百分比、完成任务数量以及具体的任务列表。

### 使用场景
- 回顾历史日期的任务完成情况
- 追踪任务完成趋势
- 查看某天具体完成了哪些任务

### 操作步骤
1. 点击首页右上角的 **"📅"** 日历按钮
2. 日历弹窗从底部弹出，默认显示当前月份和今天
3. 点击任意日期查看该日的完成情况
4. 使用左右箭头切换月份
5. 点击 **"✕"** 或空白处关闭弹窗

### 日历界面说明

#### 日历头部
- **标题**：显示 "日历"
- **关闭按钮**：点击关闭弹窗

#### 年月导航
- **左右箭头**：切换到上一个月/下一个月
- **年月显示**：如 "2026年 3月"
- **提示文字**："单击预览 · 双击跳转编辑"

#### 日期网格
- **星期标题**：日、一、二、三、四、五、六
- **日期显示**：
  - 当前月日期：正常显示
  - 其他月日期：灰色显示
  - 今天：紫色边框高亮
  - 选中日期：紫色背景
  - 有完成数据的日期：底部显示小圆点

#### 日期详情
选中日期后下方显示详细信息：
- **日期信息**：如 "3月8日 周日 · 今天"
- **完成统计**：如 "完成 19% · 勾选 3 项"
- **完成时间列表**：
  - 显示每项任务的完成时间
  - 格式：`时间  任务名称`
  - 示例：`20:51  采购食材`

#### 返回今天
- 当选中日期不是今天时，显示 **"返回今天 ↺"** 按钮
- 点击后跳回今天并刷新数据

### 数据存储
- 每日数据保存在本地存储中
- 存储键名格式：`daily_YYYY-MM-DD`
- 数据格式：
```javascript
{
  completed: 3,        // 完成任务数
  total: 25,           // 总任务数
  tasks: [             // 完成的任务列表
    { name: "任务名", time: "20:51" },
    ...
  ]
}
```

### 数据自动保存
- 每次勾选/取消勾选任务时自动保存
- 只保存当前日期的数据
- 历史数据不会被修改

### 注意事项
- 日历只显示有数据记录的日期
- 没有完成任务的日期显示 "暂无完成的任务"
- 切换日期后，任务列表实时更新
- 数据存储在本地，卸载小程序会丢失

## 技术说明

### 本地存储键名
- `adhd_tasks` - 任务数据
- `grid_size` - 布局大小设置
- `daily_YYYY-MM-DD` - 每日完成数据（如 `daily_2026-03-08`）

### 数据结构
```javascript
// 任务对象
{
  id: 时间戳,
  name: "任务名称",
  color: "背景颜色",
  completed: false
}

// 线条进度对象
{
  index: 线条索引,
  completed: 已完成数,
  total: 总格子数,
  percent: 完成百分比
}

// 日历日期对象
{
  day: 8,                    // 日期数字
  fullDate: "2026-03-08",    // 完整日期
  isCurrentMonth: true,      // 是否当前月
  isToday: false,            // 是否今天
  isSelected: true,          // 是否选中
  hasData: true,             // 是否有完成数据
  completed: 3,              // 完成任务数
  total: 25                  // 总任务数
}

// 每日完成数据
{
  completed: 3,              // 完成任务数
  total: 25,                 // 总任务数
  tasks: [                   // 完成的任务列表
    { name: "任务名", time: "20:51" }
  ]
}
```
