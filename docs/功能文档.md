## 格子布局设置功能

### 功能概述
允许用户自定义 Bingo 格子的布局大小，支持 3×3、4×4、5×5、6×6 四种布局。

### 使用场景
- 任务数量较少时选择小格子（如 3×3 = 9个任务）
- 任务数量较多时选择大格子（如 6×6 = 36个任务）
- 根据个人喜好调整界面密度

### 操作步骤
1. 点击首页底部的 **"任务"** 按钮进入任务管理页面
2. 在 **"格子布局设置"** 卡片中选择想要的布局
3. 点击 3×3、4×4、5×5 或 6×6 按钮
4. 如果当前任务数超过新布局容量，会提示是否裁剪

### 数据存储
- 布局设置保存在本地存储中，键名为 `grid_size`
- 默认布局为 5×5（25个格子）

### 注意事项
- 切换布局时，超出新布局容量的任务会被隐藏（不会删除）
- 切换回大布局时，之前隐藏的任务会重新显示
- 每个布局对应的最大任务数：
  - 3×3 = 9个任务
  - 4×4 = 16个任务
  - 5×5 = 25个任务
  - 6×6 = 36个任务

## 格子分数计算功能

### 功能概述
采用 Bingo 游戏规则计算完成进度，基于行、列、对角线的完成度来计算总体进度。

### 计算规则

#### 1. 线条定义
- **行（Row）**：横向一排格子
- **列（Column）**：纵向一排格子
- **对角线（Diagonal）**：斜向一排格子（左上到右下、右上到左下）

#### 2. 单线条完成度计算
```
单线条完成度 = (该线条已完成格子数 / 该线条总格子数) × 100%
```

**示例（4×4 布局）：**
- 一行有 4 个格子
- 完成 1 个 = 25%
- 完成 2 个 = 50%
- 完成 3 个 = 75%
- 完成 4 个 = 100%

#### 3. 总体进度计算
```
总体进度 = 所有线条完成度的平均值

线条总数 = 行数 + 列数 + 对角线数
        = n + n + 2
        = 2n + 2

其中 n 为布局大小（如 4×4 则 n=4）
```

**示例（4×4 布局）：**
- 行数：4 行
- 列数：4 列
- 对角线：2 条（主对角线 + 副对角线）
- 总线条数：4 + 4 + 2 = 10 条
- 总体进度 = 10 条线条完成度的平均值

### 进度显示
- **首页进度卡片**：显示总体完成百分比
- **进度条**：可视化展示完成进度
- **奖励弹窗**：显示完成度最高的线条信息

### 进度提示文案
根据总体进度显示不同的鼓励文案：
- 0-24%："完成一项继续加油 💪"
- 25-49%："不错的开始！继续 💪"
- 50-74%："已经完成一半了！✨"
- 75-99%："即将完成，加油！🔥"
- 100%："太棒了！今日任务全部完成 🎉"（显示庆祝动画）

### 特殊功能
- **完成庆祝**：当总体进度达到 100% 时，显示庆祝弹窗
- **线条详情**：在奖励弹窗中显示完成度最高的行/列/对角线

## 日历功能

### 功能概述
提供日历视图，用户可以查看任意日期的任务完成情况，包括完成百分比、完成任务数量以及具体的任务列表。

### 使用场景
- 回顾历史日期的任务完成情况
- 追踪任务完成趋势
- 查看某天具体完成了哪些任务

### 操作步骤
1. 点击首页右上角的 **"📅"** 日历按钮
2. 日历弹窗从底部弹出，默认显示当前月份和今天
3. 点击任意日期查看该日的完成情况
4. 使用左右箭头切换月份
5. 点击 **"✕"** 或空白处关闭弹窗

### 日历界面说明

#### 日历头部
- **标题**：显示 "日历"
- **关闭按钮**：点击关闭弹窗

#### 年月导航
- **左右箭头**：切换到上一个月/下一个月
- **年月显示**：如 "2026年 3月"
- **提示文字**："单击预览 · 双击跳转编辑"

#### 日期网格
- **星期标题**：日、一、二、三、四、五、六
- **日期显示**：
  - 当前月日期：正常显示
  - 其他月日期：灰色显示
  - 今天：紫色边框高亮
  - 选中日期：紫色背景
  - 有完成数据的日期：底部显示小圆点

#### 日期详情
选中日期后下方显示详细信息：
- **日期信息**：如 "3月8日 周日 · 今天"
- **完成统计**：如 "完成 19% · 勾选 3 项"
- **完成时间列表**：
  - 显示每项任务的完成时间
  - 格式：`时间  任务名称`
  - 示例：`20:51  采购食材`

#### 返回今天
- 当选中日期不是今天时，显示 **"返回今天 ↺"** 按钮
- 点击后跳回今天并刷新数据

### 数据存储
- 每日数据保存在本地存储中
- 存储键名格式：`daily_YYYY-MM-DD`
- 数据格式：
```javascript
{
  completed: 3,        // 完成任务数
  total: 25,           // 总任务数
  tasks: [             // 完成的任务列表
    { name: "任务名", time: "20:51" },
    ...
  ]
}
```

### 数据自动保存
- 每次勾选/取消勾选任务时自动保存
- 只保存当前日期的数据
- 历史数据不会被修改

### 注意事项
- 日历只显示有数据记录的日期
- 没有完成任务的日期显示 "暂无完成的任务"
- 切换日期后，任务列表实时更新
- 数据存储在本地，卸载小程序会丢失

## 每日自动重置功能

### 功能概述
首页任务标签在每天凌晨自动重置为默认状态（未完成）。当用户在新的一天首次打开小程序时，系统会自动检测并将所有任务的完成状态重置为未完成，让用户可以重新开始新一天的任务打卡。

### 使用场景
- 新一天开始时自动重置任务状态
- 无需手动重置昨日已完成的任务
- 保持每日任务打卡的连续性

### 工作原理
1. **日期检测**：每次打开首页时，系统会检查上次访问日期
2. **自动重置**：如果上次访问日期不是今天，则自动将所有任务状态重置为未完成
3. **状态保存**：重置后的任务状态会保存到默认任务模板和当天的任务存储中

### 数据存储
- 上次访问日期保存在本地存储中，键名为 `last_visit_date`
- 格式：`YYYY-MM-DD`（如 `2026-03-13`）
- 任务状态重置后，会同步更新以下存储：
  - `adhd_tasks` - 默认任务模板
  - `tasks_YYYY-MM-DD` - 当天的任务状态

### 技术实现
```javascript
// 检查并重置每日任务状态
checkAndResetDailyTasks() {
  const now = new Date()
  const today = `${year}-${month}-${day}`
  
  // 获取上次访问日期
  const lastVisitDate = wx.getStorageSync('last_visit_date')
  
  // 如果上次访问日期不是今天，重置任务状态
  if (lastVisitDate && lastVisitDate !== today) {
    this.resetAllTasksToDefault()
  }
  
  // 更新上次访问日期
  wx.setStorageSync('last_visit_date', today)
}
```

### 注意事项
- 重置只会在新的一天首次打开小程序时触发
- 历史完成数据（`daily_YYYY-MM-DD`）不会被删除，可用于统计回顾
- 重置后任务列表保持不变，仅完成状态变为未完成
- 如果用户在同一天内多次打开小程序，不会重复重置

## 任务分组管理功能

### 功能概述
支持创建、编辑、删除自定义任务分组，方便用户对任务进行分类管理。每个分组可以独立管理自己的任务列表。

### 使用场景
- 按生活场景分类任务（如工作、学习、家务、健康等）
- 按时间段分类任务（如早晨、下午、晚上）
- 按优先级分类任务（如紧急、重要、日常）

### 操作步骤

#### 创建新分组
1. 进入 **任务管理页面**
2. 滚动到页面底部，点击 **"+ 新建分组"** 按钮
3. 在弹窗中输入分组名称
4. 点击确定，分组创建成功后会自动显示在列表中

#### 编辑分组名称
1. 在分组标题栏点击 **"编辑"** 按钮
2. 在弹窗中修改分组名称
3. 点击确定保存修改

#### 删除分组
1. 在分组标题栏点击 **"删除"** 按钮
2. 确认删除后，该分组下的所有任务会自动移动到 **默认分组**

#### 向分组添加任务
1. 在分组底部的输入框中输入任务名称
2. 点击 **"+"** 按钮添加任务
3. 任务会自动继承分组信息

### 数据存储
- 分组信息保存在本地存储中，键名为 `task_groups`
- 分组备份保存在 `task_groups_backup`
- 分组历史记录保存在 `task_groups_history`
- 每个任务对象包含 `groupId` 和 `groupName` 字段

### 数据结构
```javascript
// 分组对象
{
  groupId: "group_1234567890",    // 分组唯一标识
  groupName: "工作任务",           // 分组名称
  createdAt: 1234567890           // 创建时间戳
}

// 带分组的任务对象
{
  id: 时间戳,
  name: "任务名称",
  color: "背景颜色",
  completed: false,
  groupId: "group_1234567890",    // 所属分组ID
  groupName: "工作任务"            // 所属分组名称
}
```

### 注意事项
- 默认分组不可删除和编辑
- 删除分组时，该分组下的任务会保留并移动到默认分组
- 分组名称不能为空
- 分组创建后立即显示在列表中

## 马卡龙色系配色方案

| 背景色 | 文字色 | 色值 | 适用场景 |
|--------|--------|------|----------|

| 淡绿色 | 深绿色 | `#E8F8F0` → `#52B788` | 健康类任务 |
| 淡黄色 | 深黄色 | `#FFF8E7` → `#E8A838` | 生活类任务 |
| 淡紫色 | 深紫色 | `#F0E8F8` → `#9B6BC3` | 休闲类任务 |
| 淡粉色 | 深粉色 | `#FCE8F0` → `#E85A8F` | 娱乐类任务 |
| 淡橙色 | 深橙色 | `#FFF0E8` → `#E8835A` | 其他类任务 |
| 淡青色 | 深青色 | `#E8F0F8` → `#5A8FE8` | 学习类任务 |
| 淡黄绿 | 深黄绿 | `#F0F8E8` → `#8FB85A` | 健康类任务 |

### 技术实现
- 背景色和文字色通过 `getTextColor()` 函数自动匹配
- 支持新旧配色方案的兼容
- 文字颜色根据背景色自动计算，确保对比度

### 视觉效果
- 柔和的背景色营造轻松愉悦的氛围
- 深色的文字确保良好的可读性
- 整体风格符合多巴胺主题的积极感受

## 用户登录功能

### 功能概述
提供用户登录注册功能，支持手机号验证码登录和微信快捷登录两种方式。当前版本为体验模式，所有功能均可免费使用，登录注册功能暂不开放。

### 使用场景
- 保存用户数据到云端
- 多设备同步任务数据
- 保护个人任务隐私

### 登录方式

#### 手机号验证码登录（预留功能）
1. 输入11位手机号码
2. 点击获取验证码（60秒倒计时）
3. 输入6位验证码
4. 点击登录/注册按钮

#### 微信快捷登录（预留功能）
1. 点击"微信快捷登录"按钮
2. 授权获取微信用户信息
3. 自动完成登录

### 体验模式
当前版本提供体验模式：
- 无需登录即可使用所有功能
- 数据保存在本地存储
- 卸载小程序后数据会丢失

### 云函数支持
- **sendSMS**：发送短信验证码
  - 生成6位随机验证码
  - 限制同一手机号1分钟内只能发送一次
  - 验证码5分钟有效
  - 支持阿里云短信服务（生产环境）
  
- **verifyCode**：验证短信验证码
  - 查询最新未使用的验证码
  - 检查验证码是否过期
  - 验证成功后标记为已使用

### 数据存储
- 用户信息保存在本地存储中，键名为 `user_info`
- 注册用户列表保存在 `registered_users`

## 周热力图功能

### 功能概述
以周为单位展示任务的完成情况，通过热力图形式直观显示一周内每天各任务的打卡状态，帮助用户追踪任务执行规律。

### 使用场景
- 查看本周任务完成情况
- 回顾历史周的任务执行记录
- 发现任务完成的时间规律
- 识别未坚持的任务

### 操作步骤
1. 进入 **统计页面**
2. 查看 **周热力图** 卡片
3. 使用左右箭头切换上一周/下一周
4. 点击日期范围可打开日历选择器快速跳转

### 界面说明

#### 周导航栏
- **左箭头**：切换到上一周
- **右箭头**：切换到下一周（当前周时禁用）
- **日期范围**：显示当前周的起止日期（如 03-09 - 03-15）

#### 热力图表头
- **任务列**：左侧显示任务名称
- **星期行**：显示周一至周日的日期（如 一 3/10）
- **今天标记**：今天的列会高亮显示

#### 热力图单元格
- **已完成**：显示任务对应的背景色
- **未完成**：显示灰色背景
- **未来日期**：显示更浅的灰色（不可打卡）

### 数据说明
- 数据从本地存储 `tasks_YYYY-MM-DD` 读取
- 显示所有当前任务列表中的任务
- 自动匹配历史日期的任务完成状态
- 未来日期自动标记为不可打卡

### 技术实现
- 周数据按周一到周日计算
- 自动计算当前周并高亮今天
- 支持通过日历选择器跳转到任意周

## 习惯榜单功能

### 功能概述
展示任务的打卡排行榜，按最近30天的完成次数排序，显示每项任务的总完成次数和连续打卡天数，帮助用户了解自己的习惯养成情况。

### 使用场景
- 查看哪些任务完成得最好
- 了解任务的坚持情况
- 发现需要加强的习惯
- 激励自己保持连续打卡

### 界面说明

#### 榜单列表
每项任务显示：
- **排名序号**：1、2、3... 表示排名
- **任务名称**：任务的名称
- **连续天数**：连续打卡天数（如"连续 5"）
- **总计次数**：最近30天总完成次数（如"总计 12"）

#### 排序规则
- 按最近30天完成次数降序排列
- 完成次数相同则按连续天数排序

### 数据计算

#### 统计范围
- 统计最近30天的打卡数据
- 从本地存储 `tasks_YYYY-MM-DD` 读取每日数据

#### 连续天数计算
- 从今天往前计算连续完成的天数
- 中断后重新计算

#### 总次数计算
- 最近30天内任务完成的累计次数

### 数据结构
```javascript
// 榜单任务对象
{
  id: 1234567890,        // 任务ID
  name: "任务名称",       // 任务名称
  color: "#E8F4FD",      // 任务颜色
  total: 12,             // 最近30天完成次数
  streak: 5,             // 连续打卡天数
  percentage: 80         // 相对最高完成次数的百分比
}
```
